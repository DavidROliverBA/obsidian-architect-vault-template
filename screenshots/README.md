# Screenshots Guide

This directory contains screenshots for the main README. To generate these screenshots, open the vault in Obsidian and capture the following views:

## Required Screenshots

### 1. `dashboard.png` - Dashboard Overview
**What to capture:**
- Open: `Dashboard - Dashboard.md`
- Show: Full dashboard with navigation links, task summary, active projects, recent meetings, and statistics
- Dimensions: ~1920x1080 (full window)
- Highlight: The central hub layout with Dataview queries rendered

**Key elements to include:**
- Quick Navigation section
- Open Tasks by priority
- Active Projects list
- Recent Meetings
- Statistics (note counts by type)

## Screenshot Guidelines

**Format:**
- PNG format (high quality)
- Standard resolution: 1920x1080 or 2560x1440 (retina)
- File size: Optimize to <500KB each if possible

**Theme:**
- Use default Obsidian theme or a professional light theme
- Ensure good contrast and readability
- Consistent theme across all screenshots

**Window:**
- Full Obsidian window (not cropped)
- Hide personal system UI (dock, menu bar) if possible
- Focus on vault content

**Content:**
- Ensure example notes are visible (not your real data!)
- Dataview queries should be rendered (not showing code)
- Use reading view where appropriate

---

## Tools for Screenshots

**macOS:**
- Cmd + Shift + 4 → Space → Click window (captures window with shadow)
- Cmd + Shift + 3 (full screen)

**Windows:**
- Win + Shift + S (Snipping Tool)
- Alt + PrtScn (active window)

**Linux:**
- Flameshot, Spectacle, or built-in screenshot tools

**Optimization:**
- [ImageOptim](https://imageoptim.com/) (macOS)
- [TinyPNG](https://tinypng.com/) (web-based)
